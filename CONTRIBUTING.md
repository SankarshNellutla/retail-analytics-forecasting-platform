Thanks for contributing! Keep contributions small and focused.

- Open an issue to discuss changes.
- Create a feature branch: `git checkout -b feat/short-description`.
- Run tests / notebooks locally.
- Keep secrets out of commits (.env is ignored).
- Use descriptive commit messages, e.g.:
  - `feat: add transform_stage logic`
  - `fix: handle missing dates in transform`
  - `chore: add .env.example and gitignore`
